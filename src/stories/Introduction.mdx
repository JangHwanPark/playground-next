# Playground Next Storybook 안내
Playground Next에서 제공하는 UI는 세 가지 층위로 나누어 스토리북에 정리되어 있습니다.
모든 컴포넌트는 기본적으로 스타일이 없는 원시 상태를 유지하므로, 전역 CSS나 디자인 시스템에서 원하는 방식으로 꾸밀 수 있습니다.

## 원시 UI (Primitive)
아무 디자인이 적용되지 않은 기본 컴포넌트입니다. **Avatar**, **Select**, **Accordion**처럼

상호작용과 접근성만 준비된 채로 제공되므로, 클래스나 인라인 스타일을 자유롭게 덧입혀 사용할 수 있습니다.

## 디자인 시스템 적용 UI (Themed)
프로덕트 전반에서 공통으로 사용하는 디자인 토큰이나 전역 스타일을 입혀 재사용할 레이어입니다.

현재는 원시 UI를 바로 사용하도록 제공하지만, 전역 `globals.css` 또는 별도 테마 파일에서 스타일을 정의해 이 섹션에 스토리를 추가하면 됩니다.

## 도메인 UI (Domain)
특정 도메인 요구사항에 맞춘 UI 조각을 배치하는 공간입니다.

예를 들어 인증, 결제, 콘텐츠 작성 등 업무별 컴포넌트를 원시/디자인 시스템 레이어 위에 얹어 구성할 수 있습니다.
이 레이어에서도 가능한 한 로직과 스타일을 분리해 유지보수를 쉽게 만드세요.
